# eIQ model — pretrained + drop-in slot

This folder ships the **pretrained eIQ Time Series Studio model** for the demo,
and doubles as the drop-in slot for your own: the connect build links whatever
package is here automatically, replacing the RMS-threshold heuristic.

| File | What |
|---|---|
| `libtss.a` | the generated static library (exports `tss_get_task_ops`; the tree data is embedded) |
| `TimeSeries.h` | the C API (`init` / `predict`) |
| `algorithm.dat`, `metadata.json` | model data + metadata (reference — accuracy, confusion matrix, build flags) |

## The shipped model

**4-class Random Forest** (`idle` / `balanced` / `unbalanced` / `both`) trained
on this demo's own motor-driven capture (see [`../training/`](../training/)):
balanced accuracy 76.55 %, ROC AUC 0.923, **RAM 0.30 KB / Flash ≈276 KB /
0.05 ms** per inference on the Cortex-M33. It classifies **per sample** (3
floats: `ax ay az` in g → class index + probabilities); the firmware runs it on
every sample of a ~1 s window and majority-votes for a stable `vib.state`
(`balanced`/`idle` → healthy, `unbalanced`/`both` → fault). Known weakness:
`both` overlaps `balanced` (recall 0.44) — expected physically, and mapped to
`fault` either way. Its only external deps are `memcpy`/`memset`, so it links
into Zephyr with no extra glue. **It is hard-float** — `connect.conf` sets
`CONFIG_FPU=y` to match.

## Regenerating (train your own)

1. Capture labeled data (Phase 1) and convert it with `tools/tss_convert.py`.
2. In TSS: import the per-class files, train an **n-Class Classification** model.
3. **Deployment** → Type **Library (.a)**, CPU **cortex-m33**, Toolchain **GCC**,
   accelerators **off** (a CML/RForest model does not use the Neutron NPU),
   `-float-abi` **hard**, `-fshort-enums` **on**, `-fshort-wchar` **off** →
   **Generate**, and unzip the package over this folder.

The full walkthrough with screenshots is in [`../QUICKSTART.md`](../QUICKSTART.md).
